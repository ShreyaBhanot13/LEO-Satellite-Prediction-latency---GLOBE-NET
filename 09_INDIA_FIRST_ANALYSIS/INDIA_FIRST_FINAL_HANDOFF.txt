================================================================================
LIMITATIONS TO STATE CLEARLY
================================================================================

1. The India regression model remains weak (R² about 0.08), so exact latency
   prediction should not be over-claimed.

2. The binary high-latency classifier is better than regression for screening,
   but it has low precision and will raise many false alarms.

3. The zone-level high-risk classifier is the best India-first prediction task,
  but it is a zone-risk classifier, not an exact latency predictor.

4. The TRAI Karnataka file currently includes only AIRTEL, so no operator-level
   benchmarking or provider ranking is valid yet.

5. The Europe-schema model remains useful only as a benchmark pipeline and not
   as the primary India model.

================================================================================
RECOMMENDED FINAL REPORT POSITIONING
================================================================================

1. Present India as the main domain of analysis.
2. Present zone-level high-risk internet performance classification as the best India-first predictive task.
3. Present direct latency regression as informative but limited.
4. Emphasize explainability and anomaly detection as strong supporting contributions.
5. Describe Karnataka as a justified case study rather than the worst-performing state.
6. Explicitly note that richer raw inputs such as operator, time, signal, or
   technology would likely improve future model quality.
