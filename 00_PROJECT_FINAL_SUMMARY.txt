This repository now contains two distinct but related analysis tracks:

1. Legacy Europe-schema optimization track
   - Purpose: model optimization on the original Europe-compatible dataset
   - Final result: CatBoost V5, R² = 0.8140
   - Role in final project: benchmark pipeline and reference implementation

2. India-first analysis track
   - Purpose: make India the primary modeling and anomaly-analysis domain
   - Data basis: original Ookla India state-labelled data plus Karnataka TRAI context

The final repository position should treat the India-first workflow as the main
project outcome and the Europe V5 workflow as a benchmark and methodology anchor.

================================================================================
TOP-LEVEL CONCLUSION
================================================================================

The strongest final contribution of this repository is not exact latency
regression alone. On the India-native workflow, direct latency regression was
weak, but explainability, high-latency screening, and dual-signal anomaly
detection produced useful and defensible findings at state level.

In the final report, the project should therefore be framed as an explainable
ML system for network-performance understanding and anomaly identification,
rather than as a high-precision latency predictor for India.

INDIA ANOMALY DETECTION - STRONGEST PRACTICAL OUTPUT
================================================================================

Primary files:
  09_INDIA_FIRST_ANALYSIS/03_outputs/india_test_predictions_with_anomaly_flags.csv
  09_INDIA_FIRST_ANALYSIS/03_outputs/india_state_anomaly_summary.csv
  09_INDIA_FIRST_ANALYSIS/03_outputs/india_priority_anomalies.csv
  09_INDIA_FIRST_ANALYSIS/04_reports/phase5_india_anomaly_detection.txt

Method:
  1. Predict expected latency with the India XGBoost regression model
  2. Compute residuals (actual - predicted)
  3. Flag residual underperformance at P95 and severe residuals at P99
  4. Combine that with the P90 high-latency classifier signal
  5. Treat overlap as dual-signal anomalies

Key India anomaly metrics:
  Test rows evaluated: 112,506
  Residual underperforming cases: 5,626
  Severe residual cases: 1,126
  Dual-signal anomalies: 3,136
  Critical anomalies: 663
  Mean residual: 5.6070 ms

Top anomaly states by dual-signal rate:
  1. Andaman and Nicobar  -> 24.64%
  2. Bihar                -> 12.30%
  3. Arunachal Pradesh    -> 10.58%

Why this matters:
  This anomaly layer is the strongest operational output of the India-first
  workflow because it identifies where actual network behavior is worse than the
  model would expect from throughput, usage, and spatial context.

================================================================================
LIMITATIONS TO STATE CLEARLY
================================================================================

1. The India regression model remains weak and should not be presented as a
   high-accuracy latency predictor.

2. The India binary high-latency classifier is useful for screening, but low
   precision means it produces many false alarms.

3. The Karnataka TRAI file currently has only one operator, so provider ranking
   and cross-operator benchmarking remain incomplete.

4. The Europe-schema model is not directly compatible with the India data and
   should only be described as a benchmark track.

================================================================================
RECOMMENDED FINAL REPORT POSITIONING
================================================================================

1. Present India as the main domain of analysis.
2. Present the Europe V5 workflow as a benchmark optimization pipeline.
3. Describe exact India latency regression as limited but informative.
4. Emphasize SHAP, high-latency screening, and anomaly detection as the stronger
   contributions of the India-first workflow.
5. Present Karnataka as a meaningful case-study state rather than the worst
   network region in India.
6. Explicitly state that richer variables such as operator, time, signal, or
   technology would likely improve future model quality.
